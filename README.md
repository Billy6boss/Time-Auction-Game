# Time-Auction-Game
時間拍賣(Time Auction Game) 是一個多人心裡遊戲，用你有的時間去與大會的時間與其他人競標，問題是你不會知道對方開價多少

## 遊戲規則
1. 房間設定每個玩家一開始擁有的時間 有 1/5/10/15/20 分鐘，並且設定回和數 5/10/15/20/25 回合
2. 每個玩家加入房間後都會有一個按鈕
3. 當每個玩家都按住按鈕後，遊戲會倒數三秒，如過在這三秒內有玩家放開按鈕，則該玩家不參與該回合本次進標
4. 三秒倒數結束後，代表遊戲(該回合)開始，房間的時間變會開始從0開始往上增加計數，最後一個放開按鈕的玩家獲勝，並獲得一分，該回合結束
5. 每個玩家需要扣除從遊戲開始到放開按鈕的時間，從自己剩下的時間中扣除，當玩家的時間扣完後，即使按住按鈕也不會參與競標
6. 回到第3點繼續下一回合，直到回合結束，最後獲得分數最高的玩家獲勝

## 功能說明
1. 每個進入之前需要幫自己的玩家取一個名字
2. 有人可以創建房間，設定房間名稱，並且設定每個玩家一開始擁有的時間和回合數
3. 房主可以決定甚麼時候開始遊戲
4. 房主開始遊戲之後，所有玩家畫面便會顯示一個按鈕，並且開始遊戲規則的流程

## 遊戲規則補充說明

- 所有玩家按住按鈕 → 伺服器啟動 3 秒倒數
- 倒數中放開 → 該玩家本回合不參與競標
- 回合開始後最後一個放開的玩家得 1 分（以伺服器接收到 ReleaseButton 的順序判定）
- 時間歸零的玩家：按鈕可按但無效，留在房間觀戰
- 若所有玩家時間歸零 → 直接結束遊戲（後續回合取消）
- 若所有玩家在倒數中放開 → 本回合平手，無人得分
- 玩家斷線 → 視為放開按鈕，從回合中移除
- 房主斷線 → 房間直接解散
- 回合結束後回到等待室，房主決定下一回合開始時間
- 所有回合結束後顯示最終得分，房主可再次開始新遊戲

## 技術說明

| 項目 | 決策 |
|---|---|
| 架構 | ASP.NET Core 10 + Razor Pages + SignalR |
| 資料儲存 | In-Memory（ConcurrentDictionary，無資料庫） |
| 玩家身份 | Cookie（關閉瀏覽器消失，不可改名） |
| 房間 ID | 6 碼隨機英數字短碼，可手動輸入分享 |
| 房間人數上限 | 30 人 |
| 最少開始人數 | 1 人（方便測試） |
| 計時顯示 | Client 端 `Date.now()` 計算，伺服器不廣播 Timer |
| 勝負判定 | 伺服器接收順序，最後收到 ReleaseButton 者獲勝 |
| 七段顯示器 | DSEG7 Classic Web Font |
| 部署 | Docker（multi-stage build），部署至 Render 免費方案 |

## 畫面
1. 當使用者進入網站後，如過沒有相關的登入資訊，則會顯示一個輸入框讓使用者輸入自己的名字
2. 當登入者進到網頁會進入一個大廳，畫面會有一個按鈕讓使用者創建房間，並且會有一個列表顯示目前所有的房間，使用者可以選擇加入房間（透過列表按鈕或輸入 6 碼短碼）
3. 當使用者加入房間後，會顯示房間的資訊，包含房間名稱、每個玩家一開始擁有的時間、回合數、目前有哪些玩家加入了房間
4. 當房主按下開始遊戲的按鈕後，所有玩家的畫面會顯示一個巨大按鈕，並且有一個時間顯示器，是經典的七段顯示器，負責顯示遊戲開始的倒數、以及遊戲開始後的時間計數
5. 該回合結束之後會回到第3點的畫面，並且顯示目前的分數，房主可以決定何時開始下一回合，直到回合結束，最後顯示獲勝者
