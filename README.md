# Time-Auction-Game
時間拍賣(Time Auction Game) 是一個多人心裡遊戲，用你有的時間去與大會的時間與其他人競標，問題是你不會知道對方開價多少

## 遊戲規則
1. 房間設定每個玩家一開始擁有的時間 有 1/5/10/15/20 分鐘，並且設定回和數 5/10/15/20/25 回合
2. 每個玩家加入房間後都會有一個按鈕
3. 當每個玩家都按住按鈕後，遊戲會倒數三秒，如過在這三秒內有玩家放開按鈕，則該玩家不參與該回合本次進標
4. 三秒倒數結束後，代表遊戲(該回合)開始，房間的時間變會開始從0開始往上增加計數，最後一個放開按鈕的玩家獲勝，並獲得一分，該回合結束
5. 每個玩家需要扣除從遊戲開始到放開按鈕的時間，從自己剩下的時間中扣除，當玩家的時間扣完後，即使按住按鈕也不會參與競標
6. 回到第3點繼續下一回合，直到回合結束，最後獲得分數最高的玩家獲勝

## 功能說明
1. 每個進入之前需要幫自己的玩家取一個名字
2. 有人可以創建房間，設定房間名稱，並且設定每個玩家一開始擁有的時間和回合數
3. 房主可以決定甚麼時候開始遊戲
4. 房主開始遊戲之後，所有玩家畫面便會顯示一個按鈕，並且開始遊戲規則的流程

## 技術說明
1. 後端：使用 .NET 10 開發，並使用 SignalR 實現即時通訊
2. 前端：使用 Razor Pages + Vanilla JS，輕量且與後端整合簡單
3. 資料存儲：使用 In-Memory 記憶體存儲，不需要資料庫，遊戲結束後資料即丟棄
4. 部署：使用 Docker（multi-stage build）部署至 Render

## 畫面
1. 當使用者進入網站後，如過沒有相關的登入資訊，則會顯示一個輸入框讓使用者輸入自己的名字
2. 當登入者進到網頁會進入一個大廳，畫面會有一個按鈕讓使用者創建房間，並且會有一個列表顯示目前所有的房間，使用者可以選擇加入房間
3. 當使用者加入房間後，會顯示房間的資訊，包含房間名稱、每個玩家一開始擁有的時間、回合數、目前有哪些玩家加入了房間
4. 當房主按下開始遊戲的按鈕後，所有玩家的畫面會顯示一個巨大按鈕，並且有一個時間顯示器，是經典的七段顯示器，負責顯示遊戲開始的倒數、以及遊戲開始後的時間計數
5. 該回合結束之後會回到第3點的畫面，並且顯示目前的分數，房主可以決定何時開始下一回合，直到回合結束，最後顯示獲勝者


## 本機開發

### 環境需求
- [.NET 10 SDK](https://dotnet.microsoft.com/download/dotnet/10.0)

### 執行步驟
```bash
# 還原套件
dotnet restore

# 啟動開發伺服器
dotnet run --project src/TimeAuctionGame
```

開啟瀏覽器前往 `https://localhost:5001`

## 部署說明（Render）

本專案使用 Dockerfile（multi-stage build）部署至 Render。

### 部署流程
1. 將程式碼推至 GitHub
2. 在 Render 建立新的 **Web Service**，選擇 Docker 執行環境
3. Render 會自動偵測 `Dockerfile` 並進行 build & deploy

### 注意事項
- 使用 **免費方案**：服務閒置約 15 分鐘後會進入 sleep 狀態，下次有請求時需約 30 秒喚醒，期間 SignalR 連線會中斷，適合個人測試與 Demo 使用
- In-Memory 資料在服務重啟後會清空，房間與遊戲紀錄不會保留
